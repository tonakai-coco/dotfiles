# 観点テンプレート

`list_activity_types.py` の出力 (実際の種別名) を必ず正として参照する。以下は典型的な日本語ロケールの Notion 監査ログで観測された種別とその対応観点。Notion のアップデートで揺れるので、未知の種別が出てきたら **意味から** 観点に振り分けてよい。

## 外部公開・共有設定の拡大
- ページのアクセス許可を更新
- データソースのページレベルのアクセスルールが更新されました
- データベースがページを作成できる権限が更新されました
- ページがWeb公開された / 共有リンクが作成された 等

## 削除・アーカイブ操作 (破壊的操作)
- ページをゴミ箱に移動
- ページをアーカイブしました
- データソースがゴミ箱に移動しました
- ファイルを削除
- 文字起こしブロックから文字起こしを削除しました
- ※ 「ページを完全に削除」「ページをゴミ箱から削除」「アクセス不可」は大半が Notion App / Notion AI / Automation 等システム自動処理。`system_actors` で除外されることでバースト誤検知は防げるが、**観点に含めると件数が多すぎてノイズになる** のでデフォルトでは含めない判断もありうる。

## 不正ログイン・認証
- ログイン
- ログアウト
- 多重失敗・MFA challenge 系の種別が出てきたら追加

## 権限変更 (管理者・ゲスト・グループ・チームスペース)
- メンバーのロールを変更
- グループにメンバーを追加 / グループからメンバーを削除 / グループ名を変更しました
- チームスペースにメンバーを追加 / チームスペースのゲスト無効設定を切り替え / チームスペースを作成
- ゲストを削除 / ゲストを招待

## 認証・セキュリティ設定 (SAML / SCIM / MFA / IP制限)
- SAML SSO 設定の有効化/無効化
- SCIM プロビジョニング設定変更
- 2要素認証ポリシー変更
- IP allowlist 設定変更

## API・インテグレーション・AIエージェント・MCP
- 外部/パブリックインテグレーションを接続
- カスタムエージェントが公開されました / 作成されました
- エージェント作成設定が更新されました
- エージェントが閲覧したウェブURL
- ※ MCP は **アクティビティ種別ではなく `MCPリダイレクトURL` 列が `N/A` 以外** で判定する

## 運用設定 (監査・分析設定)
- ユーザーアナリティクスの追跡設定を切り替え
- 監査ログ保持期間の変更
- 監査ログエクスポート

## デフォルトで除外候補
- ページを編集 / コメント追加 / 提案 (suggest) / 通常閲覧 — 件数が膨大で監査ノイズ
- ページのエクスポート / ファイルのダウンロード — 業務上の通常操作。ユーザーが明示的に含めたい場合のみ追加
- ページ認証ステータス変更 — 重要度低と判断されることが多い
