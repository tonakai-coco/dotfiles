# Git Commit (日本語)

変更内容を確認し、日本語でコミットメッセージを作成してコミットするワークフロー。

## ワークフロー

### 1. 変更内容の確認

以下のコマンドを並列実行して現在の状態を把握:

```bash
git status
git diff --staged
git diff
git log --oneline -5
```

### 2. コミットメッセージの作成

変更内容を分析し、以下の形式で日本語コミットメッセージを作成:

```
<type>: <概要>

<詳細（必要な場合）>
```

**typeの種類:**
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `style`: コードの意味に影響しない変更（空白、フォーマット等）
- `refactor`: バグ修正や機能追加を伴わないコード変更
- `test`: テストの追加・修正
- `chore`: ビルドプロセスやツールの変更

### 3. ユーザー確認

コミット前に必ずAskUserQuestionツールで確認を取る:
- 変更ファイルの一覧を提示
- 提案するコミットメッセージを表示
- ユーザーが承認するまでコミットしない

### 4. コミット実行

ユーザーの承認後、以下を実行:

```bash
git add <対象ファイル>
git commit -m "$(cat <<'EOF'
<コミットメッセージ>
EOF
)"
```

## 注意事項

- ステージングされていない変更がある場合、どのファイルをコミットするか確認する
- .envなどの機密ファイルはコミットしない
- 変更がない場合は「コミットする変更がありません」と伝える
